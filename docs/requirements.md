# 要件定義書

## プロジェクト概要

事業部キックオフイベントで利用する、行動宣言投稿・リアルタイム表示アプリ。
一時利用を目的とした使い捨てアプリケーション。

---

## イベント概要

| 項目 | 内容 |
|---|---|
| イベント日 | 2026年4月10日 |
| 想定参加人数 | 最大400名弱 |
| 利用デバイス | PC・スマートフォン（両対応） |

---

## 機能要件

### 参加者機能

| # | 機能 | 詳細 |
|---|---|---|
| F-01 | 行動宣言の投稿 | テキスト自由入力、最大140文字 |
| F-02 | 1人1投稿制限 | 同一ブラウザセッションからの重複投稿を防止 |
| F-03 | 匿名投稿 | 個人を特定する情報は不要 |
| F-04 | リアルタイム表示 | 投稿された行動宣言をワードクラウドでリアルタイム更新 |

### 管理者機能

| # | 機能 | 詳細 |
|---|---|---|
| F-05 | 投稿一覧表示 | 全投稿の一覧表示（テキスト・キーワード・投稿日時） |
| F-06 | 投稿削除 | 不適切な投稿の削除（テスト投稿の削除にも利用） |
| F-07 | CSVエクスポート | イベント終了後に全投稿内容をCSV形式でダウンロード |

### AI機能

| # | 機能 | 詳細 |
|---|---|---|
| F-09 | キーワード自動抽出 | 投稿テキストからAIがキーワードを3〜6個自動抽出 |
| F-10 | ワードクラウド生成 | 抽出キーワードの出現頻度に応じてワードクラウドを描画 |

---

## 非機能要件

| 項目 | 要件 |
|---|---|
| 同時接続 | 最大400名が同時アクセスできること |
| リアルタイム性 | 投稿後、数秒以内にワードクラウドへ反映されること |
| レスポンシブ | PC・スマートフォン両方で正常に動作すること |
| コスト | 運用コストを最小限に抑えること（無料枠の活用） |
| 可用性 | イベント当日に安定して動作すること |

---

## 制約・前提条件

- 一時利用のため、長期的な保守・拡張性は考慮しない
- 1人1投稿の制限はブラウザのsessionStorageによる簡易制御（厳密な制限ではない）
- 管理者画面は認証なし（URLを知っている運営者のみが利用する運用で対応）
- カテゴリはAIが自動判定するため、事前定義は行わない

---

## CSVエクスポート仕様

エクスポートされるCSVのカラム構成：

| カラム名 | 内容 |
|---|---|
| id | 投稿ID |
| text | 行動宣言本文 |
| keywords | AIが抽出したキーワード（カンマ区切り） |
| created_at | 投稿日時 |
